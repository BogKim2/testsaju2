# 01 — CTO · 제품 전략 (실행 결과)

> 원본 지시: `prompts/01_office_hours.md`

## MVP 현재 구현 범위 (`saju_gstack`)

아래는 코드 기준 **현재 제품**과 본 문서의 정합을 맞추기 위한 단락이다. 자동 검증 요약은 [00_gstack_sprint_execution.md](00_gstack_sprint_execution.md) 를 본다.

- **음력:** API는 `calendar=lunar` 를 거절(400)한다. 문서의 “음양” 중 **양력 MVP**만 지원한다. 음력 변환은 후속.
- **출생지·타임존:** 요청에 `timezone`(IANA), `location`(선택)을 받는다. `timezone`은 검증 후 메타에 기록하고, **원국 간지 계산은 생년월일·시 정수(벽시계)** 로만 한다. 역사적 DST 보정·출생지→IANA 자동 매핑은 MVP 비포함.
- **올해 운세(규칙):** `year_fortune` JSON(해당 연도 연주 간지, 일간 대비 연간 십성)을 **엔진이 결정적으로** 채우고, LLM은 이 필드와 원국 JSON만 해석한다.
- **자시·날짜 경계:** 자시(23–01) 일자 규칙은 별도 문서화 전까지 엔진의 시지 조견표만 사용한다.

## 1. 제품 스코프

### 포함 (MVP 핵심)

- 생년월일시·음양·성별·(선택) 이름·출생지 기반으로 **원국 사주(년·월·일·시 천간지지)** 를 Python으로만 계산한다.
- 그 결과로 **오행 분포·일간·십성 구조**를 같은 엔진에서 산출한다.
- 위 결과를 **구조화 JSON**으로 고정한 뒤, LLM은 **한국어 해석·요약·영역별 문장**만 생성한다.
- **올해 운세**는 규칙 기반으로 산출 가능한 범위(예: 년운 요약 한 블록)만 다룬다.

### 명시적 제외 (초기)

- 합·충·형·파·공망 등 세밀 신살·택일·궁합 풀옵션.
- 실시간 채팅형 무한 상담, 타인 사주 비교 등 소셜·커뮤니티.
- 법적·의료적 조언으로 오해될 수 있는 단정적 진단 문구.

---

## 2. 리스크 (환각 · 문화 정확성)

| 리스크 | 내용 |
|--------|------|
| 환각 | LLM이 천간·지지·오행 수치를 스스로 추정하거나 JSON과 불일치하는 서술을 할 때 발생한다. |
| 일관성 | 같은 JSON인데 실행마다 톤·결론이 달라지면 신뢰가 무너진다. |
| 문화·학파 | 사주는 파별·해석 차이가 크다. “유일한 정답”처럼 쓰면 반발이 생긴다. |
| 음력·시간대 | 음양 전환·자시 처리·타임존 오류 시 원국 자체가 틀어진다 (LLM과 무관한 치명적 버그). |
| 민감 주제 | 건강·재물·연애를 단정하면 법적·윤리적 이슈가 된다. |

**대응:** 계산은 전부 백엔드 고정 로직, LLM 입력은 검증된 JSON만, 프롬프트에 “계산 금지·데이터 외 추측 금지·경향어 사용”을 명시한다.

---

## 3. MVP 경계

**반드시:** 단일 플로우 입력 → 분석 API → 결과 화면(표 + 요약 + LLM 텍스트). 음/양, 성별, 시간(미입력 시 정책 1가지 문서화). 올해 운세는 짧은 요약 1블록.

**이번 버전에서 제외:** 회원·결제·저장·공유 고도화, 다국어, 앱 스토어, 복수 학파 선택 UI, LLM A/B.

**성공 기준:** 동일 입력 → **사주 JSON 동일**, 해석만 약간 변할 수 있으나 사실 서술은 JSON과 모순 없음.

## 4. 기존 운세 서비스 대비 차별화

1. **“계산은 코드, 말은 AI”** — 숫자·팔자표는 재현 가능하게 보여 준다.
2. **투명한 데이터** — 천간·지지·오행·십성을 먼저 보여 준 뒤 그 아래 해석을 붙인다.
3. **로컬/프라이버시** — LM Studio 로컬 추론 시 외부 LLM으로 원본이 나가지 않게 설계할 수 있다 (기술 조건 충족 시).
4. **톤** — 과장·공포 마케팅 대신 전문적이되 부드러운 한국어, 경향 중심.

## 한 줄 요약

스코프는 **“결정적 Python 사주 엔진 + JSON 기반 한국어 해석”** 으로 고정하고, 리스크는 LLM의 계산·단정·JSON 불일치를 막는 설계로 줄이며, MVP는 한 갈래 사용자 여정과 한 번의 분석 품질에 집중한다.
